1.Prototype and Prototypal Inheritance in Javascript(This video by Akshay Saini provides an in-depth explanation of Prototypes and Prototypal Inheritance, which are fundamental concepts for understanding how JavaScript works under the hood.

### **Core Concepts**

* **What is a Prototype?**
* In JavaScript, every object, function, and array is automatically attached to a hidden object known as a prototype.
* This prototype object contains various built-in properties and methods that the original object can access.
* For example, when you create an object `let arr = []`, it automatically gets access to methods like `.length`, `.push()`, and `.pop()` because they exist on its prototype.


* **The Prototype Chain**
* When you try to access a property or method on an object, JavaScript first looks at the object itself.
* If it's not found, it looks at the object's prototype (`__proto__`).
* This process continues up a "chain" of prototypes until the property is found or the chain ends at `null`.
* **The Chain Hierarchy:**
1. **Object instance** (e.g., `myObj`).
2. **`Object.prototype`** (where most standard methods like `.toString()` live).
3. **`null`** (the absolute end of the chain).




* **Prototypal Inheritance**
* Inheritance in JavaScript is the mechanism by which one object can access the variables and methods of another object.
* Unlike classical inheritance (found in languages like Java), JavaScript uses the prototype chain to achieve this.
* You can manually set an object's prototype using `__proto__` (older way) or `Object.setPrototypeOf()`.



### **Practical Examples from the Video**

* **Sharing Methods:** If you have multiple objects that need the same function, instead of defining it in every object (which wastes memory), you can define it once on a prototype that all those objects share.
* **Method Borrowing:** By setting `objectA.__proto__ = objectB`, `objectA` can now "borrow" and use any methods or properties defined in `objectB` as if they were its own.

### **Key Interview Questions Answered**

* **What is `__proto__`?** It is a reference to the prototype of the object.
* **What is the difference between `prototype` and `__proto__`?** `prototype` is a property of a constructor function, while `__proto__` is a property of an object instance that points to its prototype.
* **Why is it called "Prototypal Inheritance"?** Because the inheritance happens through the prototype objects rather than class blueprints.
