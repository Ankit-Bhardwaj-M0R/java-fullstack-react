7.call, apply and bind method(The video "call, apply and bind method in JavaScript" by Akshay Saini explains how these three methods are used to manipulate the `this` context in JavaScript functions. Below is a summary of the key concepts discussed in the video based on the provided metadata and external technical references.

### **1. The `call()` Method**

* **Purpose:** It allows you to invoke a function while explicitly specifying the value of `this` for that function call.
* **Usage:** You pass the object you want `this` to point to as the first argument. Any subsequent arguments are passed to the function normally.
* **Example Scenario:** If you have a method inside one object and you want to use it for another object without re-defining it, you can "borrow" the method using `call()`.

### **2. The `apply()` Method**

* **Purpose:** This method is very similar to `call()`, but it differs in how it handles additional arguments.
* **Usage:** Instead of passing arguments individually, you pass them as a single **array**.
* **Key Difference:** `fun.call(obj, arg1, arg2)` vs. `fun.apply(obj, [arg1, arg2])`.

### **3. The `bind()` Method**

* **Purpose:** Unlike `call` and `apply`, which execute the function immediately, `bind()` creates a **new copy** of the function.
* **Usage:** It "binds" the original function to a specific object and returns it, allowing you to call this new function later in your code.
* **Interview Tip:** This is frequently used when you want to ensure a function retains a specific context regardless of where it is eventually called (e.g., inside an event listener).

### **Key Takeaways for Interviews**

* **Function Borrowing:** Both `call` and `apply` are used for function borrowing.
* **Immediate vs. Delayed Execution:** `call` and `apply` execute the function immediately; `bind` returns a bound function for later use.
* **Arguments Passing:** `call` takes a list of arguments, while `apply` takes an array of arguments.
